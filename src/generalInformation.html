Достъпност

Диаграмите на Chart.js се рендерират чрез платна (canvas). Наша задача е да създадем този елемент по такъв начин, че той да бъде достъпен. Добрата новина е, че той се поддържа от всички браузъри, лошата - съдържанието му не е достъпно за четци на екрани. При елементът платно, достъпността трябва да бъде добавена чрез ARIA атрибути или чрез използване на резервно съдържание, което трябва да се намира между отварящите и затварящите тагове на елемента платно.

Пример за добра достъпност чрез ARIA атрибути

В този пример имаме зададени атрибутите етикет и роля, които осигуряват достъпно име на елементa платно

<canvas id="good_example_canva" width="300" height="150" aria-label="Labeled" role="img"></canvas>

Пример за добра достъпност чрез резервно съдържание

Резервното съдържание подсигурява текстова алтернатива на елементa платно

<canvas id="good_example_canva_fbc" width="300" height="150"><p>Fallback Content</p></canvas>

Пример за лоша достъпност - без ARIA атрибути

В този пример липсва достъпно име или роля за елементa платно

<canvas id="bad_example_canva" width="300" height="150"></canvas>

Пример за лоша достъпност - без резервно съдържание

Резервното съдържание на елементa платно е недостъпно

<canvas id="bad_example_canva_fbc" width="300" height="150">Bad Fallback Content</canvas>

Цветове

Библиотеката Chart.js разполага с предварително зададени цветове. Въпреки това е предоставена възможност те да бъдат настроени ръчно. За тази цел могат да бъдат използвани шестнадесетичните стойности на цветовете, тяхната стойност по RBG схемата или според HSL схемата. Поддържат се 3 основни вида промени в това отношение:

  • Промяна на цвета на фона, както и цвета на границите на геометричните обекти.

  • Промяна на цвета на текста на текстовите елементи.

  • Промяна на цвета на фона на цялото платно.

Примери за промяна на цветове

                            Chart.defaults.borderColor='red'
                            Chart.defaults.color=rgb(255, 0, 255)
                            Chart.defaults.color=#94618E

Цветове по подразбиране

Използването на цветовете по подразбиране не е особено добра практика. При наличие на няколко множества от данни те ще бъдат на практика неразличими. За да се избегне това може да се зададат различни стойности на цветовете на фона и границите на всеки един от тях. Това обаче може да означава допълнително време и работа. За улеснение могат да бъдат използвани допълнения (plugins), които разполагат с предефинирани или генерирани цветови гами. Библиотеката Chart.js има вградено допълнение Colors, което предоставя 7 цвята. За да може да се използва е необходимо да се включи и регистрира в кода.

                            import {Colors} from 'chart.js';
                            Chart.register(Colors);

Структури от данни

Подаването на свойството/полето данни може да бъде осъществено под различни формати. По подразбиране свойството/полето данни на множеството данни се преобразува в съответствие с избраната диаграма. Важно е да се отбележи, че ако бъде използвано свойството/полето етикети, то трябва да съдържа същият брой елементи като този на множеството от данни с най-много елементи. Полето/свойството етикети се използва, за да бъдат поставени означения в оста на индексите, т.е. оста Х. За да бъдат рендерирани правилно етикетите, те трябва дая бъдат от тип стрин или число.

Масив от примитивни типове

                             const cfg = {
                               type: 'bar',
                               data: {
                                    datasets: [{
                                     data: [2, 3, 4, 5, 6]
                                    }],
                                    labels: ['слаб', 'среден', 'добър', 'много добър', 'отличен']
                                   }
                                 }

Масив от обекти

Това всъщност е форматът, който се използва вътрешно за преработените данни. В този режим преобразуването може да бъде изкллючено от опциите (parsing: flase). Ако това бъде направено, данните трябва да бъдат сортирани и представени в съответния формат, който определената диаграма и мащаби използват вътрешно. Нулевата стойност (null) може да бъде използвана за пропускане на стойности.

                             const cfg = {
                               type: 'bar',
                               data: {
                                    datasets: [{
                                     data: [{x: 'Ivancho', y: 5}, {x: 'Maria', y: 6}}]
                                    }],
                                   }
                                 }

Масив от обекти с персонализирани свойства/полета

Ако използваме пай или донът диаграма е важно обектът да има ключ, който да посочва стойността. Освен това, ако ключа съдържа точка, тя трябва да се избегне с двойна наклонена черта

                             const cfg = {
                               type: 'bar',
                               data: {
                                    datasets: [{
                                     data: [{brand: 'Mercedes', price: {value: 35}}, {brand: 'BMW', price: {value: 33}}]
                                    }],
                                   }
                               options: {
                                     parsing: {
                                      key: 'price.value'
                                    }
                                   }
                                  }

Обект

В този случай името на обекта служи за индекс, а стойността става стойност.

                             const cfg = {
                               type: 'bar',
                               data: {
                                    datasets: [{
                                     data: {
                                       Издържали изпита: 60
                                       Скъсани: 13
                                      }
                                    }],
                                   }
                                 }

Шрифтове

Chart.js предлага възможност за промяна на шрифта, неговия размер, стил, тегло и т.н. По подразбиране, ако не бъдат заявени специфични опции се използват глобалните настройки. Специалните глобални настройки могат да бъдат достъпени чрез Chart.defaults.font.

Важно уточнение: Ако бъде използван шрифт, който не съществува в системата, браузърът няма да го го приложи. Ако възникнат подобни проблеми, или има наличие на странности е добре да се провери дали подадения шрифт съществува в системата.

Име Тип Стойност по подразбиране
Семейство (family) стринг "Helvetica neue", "Helvetica", "Arial", sans-serif
Размер (size) число 12
Стил (style) стринг 'normal'
Тегло (weight) normal | bold | lighter | bolder | number undefined
Височина на реда (lineHeight) число | стринг 1.2
